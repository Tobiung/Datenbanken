Das Drei-Ebenen-Schema von Datenbanksystemen

Das Drei-Ebenen-Schema (auch Drei-Schema-Architektur genannt) ist ein grundlegendes Konzept in der Datenbankentwicklung. Es teilt ein Datenbanksystem in drei verschiedene Abstraktionsebenen auf, um Datenunabhängigkeit zu gewährleisten.

Externe Sicht
(Anwendersicht / External View)
Konzeptionelle Sicht
(Logisches Schema / Conceptual View)
Interne Sicht
(Physisches Schema / Internal View)

1. Externe Sicht (External View)

Die Sicht des Anwenders, der die Datenbank mit einer Abfragesprache nutzt. Verschiedene Benutzer können unterschiedliche externe Sichten auf dieselbe Datenbank haben.

Beispiel:

Ein Buchhaltungsmitarbeiter sieht nur Finanzdaten, während ein Lagerleiter Bestandsdaten sieht.

2. Konzeptionelle Sicht (Conceptual View)

Beschreibt das Datenmodell – die datenmäßige Abbildung eines Ausschnitts der realen Welt. Bei relationalen Datenbanken werden Daten in Tabellen angeordnet und Beziehungen zwischen ihnen geregelt.

Beispiel:

Das gesamte Datenbankschema mit allen Tabellen, Attributen und Beziehungen.

3. Interne Sicht (Internal View)

Behandelt die physische Speicherung der Daten. Das DBMS regelt, wie Daten gespeichert werden und führt einen Systemkatalog (Data Dictionary).

Beispiel:

Speicherung auf Festplatte, Indexstrukturen, Speicherplatzverwaltung.

Vorteile der Unterteilung in Ebenen:

  • Datenunabhängigkeit: Änderungen auf einer Ebene beeinflussen andere Ebenen nicht. Beispiel: Die physische Speicherung kann optimiert werden, ohne dass Anwendungen angepasst werden müssen.
  • Abstraktion: Benutzer müssen sich nicht mit technischen Details befassen. Sie arbeiten mit einer für sie verständlichen Sicht der Daten.
  • Wartbarkeit: Einfachere Wartung und Erweiterung des Systems. Änderungen können isoliert auf einer Ebene vorgenommen werden.
  • Sicherheit: Gezielte Zugriffssteuerung auf verschiedenen Ebenen möglich. Benutzer sehen nur die Daten, für die sie berechtigt sind.

Zusammenfassung:

Das Drei-Ebenen-Schema ermöglicht eine klare Trennung zwischen:

  • Anwenderperspektive (externe Sicht)
  • Logischer Struktur (konzeptionelle Sicht)
  • Physischer Implementierung (interne Sicht)

Diese Trennung ist fundamental für moderne Datenbanksysteme und ermöglicht Flexibilität, Skalierbarkeit und Wartbarkeit.
